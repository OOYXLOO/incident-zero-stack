Decision in 30 seconds

Why this belongs in the Slack agent track

  • The primary user action is a Slack command: /incident-zero scenario=identity severity=critical.
  • The response is a structured Block Kit incident brief with severity, confidence, action buttons, evidence, and executive handoff metadata.
  • The same incident engine powers the public Vercel endpoint, local cockpit, static review pages, and MCP tools.
  • The product targets an operational workflow Slack teams already run during incidents: triage, containment, evidence collection, stakeholder updates, and handoff.

Agent surface

MCP handoff

  • incident_zero_brief returns the command-style incident summary.
  • incident_zero_handoff exports an executive handoff.
  • incident_zero_storage_preview shows the deterministic audit record shape.
  • The MCP server reuses the same model as the Slack endpoint instead of maintaining a separate mock.

Product impact

Why teams would use it

  • Reduces incident-channel thrash by turning noisy facts into one brief.
  • Keeps audit records structured from the first response minute.
  • Separates public proof from private workspace data and credentials.
  • Gives small teams a repeatable path from alert to executive update.

Remaining gates

What still requires the account owner

The repository intentionally does not store Slack tokens, signing secrets, workspace cookies, private messages, customer records, payment data, or credentials.

Before final submission, the account owner still needs to import the Slack manifest, install the app in a sandbox workspace, invite challenge testers, record a real Slack command clip, and submit the challenge form.
